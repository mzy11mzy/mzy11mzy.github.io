Check List

  • Run through each page with the WAVE Chrome Extension. (Failed)
  • Users should be able to navigate through content using their keyboard. (Passed)
  • Users should be able to navigate content using a screen reader. (Failed)
  • The general architecture and hierarchy of the content should make sense. (Passed)
  • Charts and images should all have alt-text so that users with screen readers or users on a slow connection will still be able to understand the images. (Failed)
  • Decorative images should not be visible to screen readers. (Passed)

Suggestions

Add alternative text to non-decorative image

In the home page, alt text of the image “Student Support”, “Engaged Learning”, and “Bold Ideas” is missing. I suggest putting the content of the image in the alt attribute so that users can understand the image given the situation that the network connection is slow or users uses a screen reader.

Add text to make content readable and understandable

In the homepage, the website uses a simple image to imitate the text and button. Therefore, the accessibility is not good, especially for the users who view the website though the screen reader. I suggest replacing the fake text and button image by the code to make it readable and understandable.

Increase the contrast between the text color and the background color

In the homepage, the color of the text in the navigation bar has a very low contrast with the background. I suggest adjusting the color of the text or background to increase the contrast.

Fix the empty button

In the homepage, the search button doesn’t provide the value txt. Therefore, screen reader users cannot indicate the function of the button. I recommend adding descriptive text within the button tags.
